# swim-training-manager
ポートフォリオのために作成しています。

## 概要（目的）
水泳トレーニングの「メニュー（計画）」と「記録（実績）」を管理し、期間別の集計・分析と、条件に基づく練習メニューの自動生成を行うWebアプリです。  
趣味を題材にしつつ、業務系システム（計画/実績管理）の設計を意識して作成しています。

## 想定ユーザー
- 個人ユーザー（将来的に複数ユーザー/チーム利用へ拡張可能な設計を想定）

## 主な機能（MVP）
- ユーザー登録 / ログイン（認証）
- 練習メニュー（計画）の作成・編集・削除
- 練習記録（実績）の登録・編集・削除
- 期間別の集計（週/月の総距離、種目別など）
- 条件に基づくメニュー自動生成（シンプルなルールベース）

## 設計方針（アピールポイント）
- 「計画（Menu）」と「実績（WorkoutLog）」を分離し、業務アプリに近いデータ構造とする
- ロジック層（Service）と表示（Controller/View）の責務分離
- 将来的な拡張（チーム/コーチ権限、目標管理、生成ルール追加）を見据えた設計
- セキュリティ・バイ・デザインの考え方を意識（認証・認可、入力値チェック、ログ）

## 技術スタック
- Java 17
- Spring Boot（Spring MVC / Spring Data JPA / Spring Security）
- Thymeleaf
- PostgreSQL
- Maven
- GitHub

## 画面（予定）
- ログイン / ユーザー登録
- メニュー一覧 / メニュー作成・編集
- 記録一覧 / 記録登録
- 集計（週/月）
